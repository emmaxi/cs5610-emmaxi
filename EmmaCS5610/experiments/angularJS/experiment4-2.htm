The purpose of this experiment is about how to use Angular data model to render array.

First of all, we create several objects in JavaScript 

    var boston = {
        city: "Boston",
        country: "USA",
        temperature: "27",
        weather: "patchy light snow",
        wind: "7",
        humidity: "86"
    };
    var beijing = {
        city: "Beijing",
        country: "China",
        temperature: "18",
        weather: "Sunny",
        wind: "0",
        humidity: "85"
    };
    var baoji = {
        city: "Baoji",
        country: "China",
        temperature: "33",
        weather: "Sunny",
        wind: "5",
        humidity: "35"
    }
    var los = {
        city: "Los Angeles",
        country: "USA",
        temperature: "84",
        weather: "Sunny",
        wind: "0",
        humidity: "11"
    }

then create an array combined by these objects above.The order in this array does not mean the order we see on the table. Since I used the filter of "temperature" to defined the new order rule. Of Course, default of the order is ascending. 

            var cities = [boston, baoji, beijing, los];

            <tbody>
              <tr ng-repeat="city in cities | orderBy:'country'">
                table here
              </tr>
            </tbody>

and give the value to objects which we want to display on the page: 

           $scope.cities = cities;

At last we use an attribution "ng-repeat" to iterate this array, render them in a table. 

           <tbody>
              <tr ng-repeat="city in cities | orderBy:'country'">
                <td>{{city.city}}</td>
                <td>{{city.country}}</td>
                <td>{{city.temperature}}</td>
                <td>{{city.weather}}</td>
                <td>{{city.wind}}</td>
                <td>{{city.humidity}}</td>
              </tr>
            </tbody>
